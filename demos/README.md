# Demos
This folder is used to hold information relating to demo's for internal and external customers. The formatting of this should folder should be: `feature > ${date}-${customer}`. This will enable us to keep a living record of any info that we use for demo's over time.

For example

```
| development-tools
|--- demos
    |--- Customizable Dashboards
        |--- 2026-04-13-corpA
        |--- 2026-04-20-corpB
    |--- Multicluster Alerting
        |--- 2026-04-13-corpA
        |--- 2026-05-02-sprint-review
```

This folder SHOULD NOT contain any sensitive customer data, and instead should only be used for our own resources for setting up and communicating about our demos. Recordings of demo's should be linked to with appropriate RBACed links within a markdown document in each folder. Demos SHOULD NOT be directly uploaded to github.
